Tutorial

Create a Discrete Global Grid System (DGGS)

Create a data cube in geographical coordinates:

using DGGS
lon_range = -180:180
lat_range = -90:90
data = [exp(cosd(lon)) + 3(lat / 90) for lon in lon_range, lat in lat_range]
geo_cube = GeoCube(data, lat_range, lon_range)
DGGS GeoCube
Element type:       Float64
Size:               510.48 KB
Axes:
lon                 Axis with 361 Elements from -180 to 180
lat                 Axis with 181 Elements from -90 to 90

using CairoMakie
plot_map(geo_cube)

Let's create a DGGS using Synder Equal Area projection (ISEA), an aperture of 4 (number of child cells of a given parent cell), a hexagonal grid shape at 3 different levels:

dggs = DgGlobalGridSystem(geo_cube, 3, :isea, 4, :hexagon)
DGGS custom DgGlobalGridSystem
Cells:   3 levels with up to 162 cells of type Union{Missing, Float64}
Grid:    DgGrid with hexagon topology, isea projection, and aperture 4
Size:    1.69 KB

The data cube at the highest level has only one spatial index dimension, i.e. the cell id:

dggs
DGGS custom DgGlobalGridSystem
Cells:   3 levels with up to 162 cells of type Union{Missing, Float64}
Grid:    DgGrid with hexagon topology, isea projection, and aperture 4
Size:    1.69 KB

Plot the DGGS at a given level

plot_map(dggs[3])

A DGGS cell represent all points within its boundary polygon. This acts as a pooling mechanism. The hexagonal topology is easily recognizable at this low level.

Create grids

Using DGGRID:

using DGGS
grid1 = DgGrid(:isea, 4, :hexagon, 3)
DgGrid custom with hexagon topology, isea projection, aperture of 4, and 642 cells at resolution 3

Using a vector of geographical coordinates for center points:

center_points = [-170 -80; -165.12 81.12; -160 90]
grid2 = Grid(center_points')
Grid with 3 cells

Explore the grid

A DGGS consists of multiple grids with varying levels. Let's create our first grid to explore its properties:

using DGGS
grid = DgGrid(:isea, 4, :hexagon, 3)
DgGrid custom with hexagon topology, isea projection, aperture of 4, and 642 cells at resolution 3

Create a data frame containing the center point or boundary polygon for all cells of the grid:

boundaries = get_cell_boundaries(grid)
centers = get_cell_centers(grid)

println(boundaries[1:5,:])
5×2 DataFrame
 Row │ geometry              cell_id
     │ IGeometr…             String
─────┼───────────────────────────────
   1 │ Geometry: wkbPolygon  1
   2 │ Geometry: wkbPolygon  2
   3 │ Geometry: wkbPolygon  3
   4 │ Geometry: wkbPolygon  4
   5 │ Geometry: wkbPolygon  5

The data frames can be saved e.g. as geojson to be used in other tools like GIS:

using GeoDataFrames
GeoDataFrames.write("boundaries.geojson", boundaries)
"boundaries.geojson"

Convert points between cell id and geographic coordinates:

get_cell_ids(grid, 80, -170)
5

and vice versa:

get_geo_coords(grid, 5)
(81.8823154, -168.75)

The coordinates may differ slightly, because a cell covers all points of a given area and only the center point is returned. This tutorial uses very low levels to demonstrate the properties of a DGGS. In practice, much higher level levels should be chosen for spatial analysis, diminishing these inaccuracies.

Import Earth System Data Cube (ESDC)

Earth System Data Lab provides global data about 42 features about temperature, precipitation, among others. The data is stored online in the Earth System Data Cube (ESDC) and can be imported with the following commands:

using DGGS
using EarthDataLab
using YAXArrays
esdc_cube = esdc(res="low")
geo_cube = GeoCube(esdc_cube)
DGGS GeoCube
Element type:       Union{Missing, Float32}
Size:               320.87 GB
Axes:
lon                 Axis with 1440 Elements from -179.875 to 179.875
lat                 Axis with 720 Elements from -89.875 to 89.875
time                Axis with 1978 Elements from 1979-01-05T00:00:00 to 2021-12-31T00:00:00
Variable            Axis with 42 elements: sensible_heat potential_evaporation .. net_ecosystem_exchange snow_sublimation 
units: mm d^-1

Plot the first variable (NDVI) at the time point:

using CairoMakie
sub_geo_cube = geo_cube[time = DateTime("2020-01-05T00:00:00"), Variable = "ndvi"]
plot_map(sub_geo_cube)

Import Zarr Arrays

Load the zarr data into a YAXArray:

using Zarr, YAXArrays
url = "gs://cmip6/CMIP6/ScenarioMIP/DKRZ/MPI-ESM1-2-HR/ssp585/r1i1p1f1/3hr/tas/gn/v20190710/"
geo_dataset = open_dataset(zopen(url, consolidated=true))
geo_array = geo_dataset["tas"]

Transform the cooordinates and create the GeoCube:

using DGGS
data = circshift(geo_array[:, :, 1], length(geo_array.lon) / 2)
latitudes = geo_array.lat
longitudes = geo_array.lon .- 180
geo_cube = GeoCube(data, latitudes, longitudes)
plot_map(geo_cube)

Import NetCDF files

Here we will explore Sea surface temperatures collected by PCMDI for use by the IPCC stored in a NetCDF file. Download the NetCDF file into a YAXArray:

using YAXArrays
using NetCDF
using Downloads
url = "https://www.unidata.ucar.edu/software/netcdf/examples/tos_O1_2001-2002.nc"
filename = Downloads.download(url, "tos_O1_2001-2002.nc")
geo_array = YAXArrays.Cube("tos_O1_2001-2002.nc")

lat                 Axis with 170 Elements from -79.5 to 89.5
time                Axis with 24 Elements from CFTime.DateTime360Day(2001-01-16T00:00:00) to CFTime.DateTime360Day(2002-12-16T00:00:00)

Lets have a look at the first time point of the raw data:

using CairoMakie
heatmap(geo_array[:,:,1])

Transform the cooordinates and create the GeoCube:

using DGGS
data = circshift(geo_array[:,:,1], 90)
latitudes = geo_array.lat
longitudes = geo_array.lon .- 180
geo_cube = GeoCube(data, latitudes, longitudes)
DGGS GeoCube
Element type:       Union{Missing, Float32}
Size:               119.53 KB
Axes:
lon                 Axis with 180 Elements from -179.0 to 179.0
lat                 Axis with 170 Elements from -79.5 to 89.5

Plot the imported geo data cube:

plot_map(geo_cube)

Since this dataset is about ocean temperature, we do not have cells on the land area.
